Implementation Requirements

  • All classes, TaxCalculator, StateTax, and TaxNavigator, should be created to represent the Taxation System.
  • Implement all instance fields and methods required to manage the taxation system.
  • Utilize the Superclass Constructor for Initializing Instance Fields in the Subclass:
      • In OOP, it is essential to utilize the constructor of the superclass when initializing instance fields in the subclass constructor. This practice ensures that the fields inherited from the superclass are properly initialized, maintaining the integrity and consistency of the object's state. By calling the superclass constructor using super(), the subclass can leverage the initialization logic defined in the superclass, which often includes setting up fundamental attributes that are crucial for the object's correct functionality. This not only promotes code reuse but also simplifies maintenance and readability, as the initialization logic is centralized in the superclass constructor. Moreover, it ensures that any changes in the initialization process in the superclass automatically propagate to the subclass, reducing the risk of errors and inconsistencies.
  • Utilize the Following Logic of the Loop Statement in the Main Method:
      • The main method of the TaxNavigator class contains a loop that processes each line of input from the "TaxData.csv" file, calculating various tax-related figures and writing the results to the "OutputTaxData.csv" file. The loop begins by reading each line of the input file, skipping the header line. For each subsequent line, the loop splits the line into individual attributes, such as state option, income, property value, and expenses. Based on the state option, it creates an appropriate StateTax object with predefined tax rates. The loop then calculates the income tax, property tax, total tax, deductions, tax return, and tax withholding using the methods defined in theTaxCalculator and StateTax classes. These calculated values are appended to the original record and written to the output file. This loop ensures that all records in the input file are processed sequentially, and the corresponding tax calculations are efficiently performed and stored in the output file.
  • Calculating tax values should not involve checking individual attributes per line and finding the tax values directly from the file data. Instead, create StateTax objects from the file data lines.

Instance Field Values for Each State

  • California:
    • State Name = "California"
    • Income Tax Rate = 0.093
    • Property Tax Rate = 0.007
    • Deduction Rate = 0.175
  • New York:
    • State Name = "New York"
    • Income Tax Rate = 0.0685
    • Property Tax Rate = 0.04
    • Deduction Rate = 0.2

Tax Calculation Formulas

  • Income Tax = Income * Income Tax Rate
  • Property Tax = Property Value * Property Tax Rate
  • Total Tax = Income Tax + Property Tax
  • Tax Deductions = Expenses * Deduction Rate
  • Tax Return = Total Tax Paid - Deductions
  • There are differences in the calculation of tax withholding between California and New York:
    • California -> Tax Withholding = Income Tax Rate * Gross Income - (allowances * 100). Assume the allowances in California is 2
    • New York -> Tax Withholding = Income Tax Rate * Gross Income - 500
